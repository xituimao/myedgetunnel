## 已完成的工作

### ✅ 1. 创建 Node.js 适配层

**文件**: `server.js`

这是核心适配文件，它做了以下工作：
- 将 Cloudflare Worker 代码适配到 Node.js 环境
- 模拟 Cloudflare 的 KV 存储（使用内存存储）
- 模拟 Cloudflare 的 `connect` API（用于网络连接）
- 处理 HTTP 和 WebSocket 请求
- 提供与 Cloudflare Worker 相同的接口

### ✅ 2. 配置项目依赖

**文件**: `package.json`

包含：
- Node.js 18+ 版本要求
- WebSocket 支持（ws 库）
- npm 脚本命令

### ✅ 3. 创建部署脚本

#### 自动部署脚本
**文件**: `deploy.sh`

功能：
- 一键自动部署到服务器
- 自动安装 Node.js 和 PM2
- 自动配置防火墙
- 自动启动应用

使用方法：
```bash
chmod +x deploy.sh
./deploy.sh
```

⚠️ 需要先安装 sshpass：
```bash
# Ubuntu/Debian
sudo apt-get install sshpass

# macOS
brew install hudochenkov/sshpass/sshpass
```

#### 手动部署引导脚本
**文件**: `deploy-manual.sh`

功能：
- 分步引导手动部署
- 不需要 sshpass
- 适合不熟悉命令行的用户

使用方法：
```bash
chmod +x deploy-manual.sh
./deploy-manual.sh
```

### ✅ 4. 进程管理配置

**文件**: `ecosystem.config.cjs`

使用 PM2 进行进程管理，提供：
- 自动重启
- 日志管理
- 集群模式支持
- 内存限制
- 开机自启

### ✅ 5. Nginx 反向代理配置

**文件**: `nginx/edgetunnel.conf`

可选配置，用于：
- 使用 80/443 端口代替 8080
- WebSocket 支持
- SSL/TLS 配置
- 负载均衡

### ✅ 6. 环境变量配置

**文件**: `.env.example`

可配置的环境变量：
- `PORT` - 服务端口（默认 8080）
- `ADMIN` - 管理员密码（必填）
- `KEY` - 快速订阅密钥
- `UUID` - 固定 UUID
- `PROXYIP` - 代理 IP
- 等等...

### ✅ 7. 完整文档

创建了三份文档：

1. **快速部署.md** - 中文快速入门指南
2. **README_DEPLOY.md** - 详细的中文部署和管理文档
3. **DEPLOY.md** - 英文部署文档

## 部署步骤

### 推荐方式：使用自动部署脚本

```bash
# 1. 确保 deploy.sh 可执行
chmod +x deploy.sh

# 2. 运行部署脚本
./deploy.sh
```

脚本会自动完成所有部署步骤，大约需要 3-5 分钟。


按照提示逐步完成部署。

如果以上方式都不可用，请参考 `README_DEPLOY.md` 中的详细步骤。

## 故障排查

### 应用无法启动

```bash
# 查看错误日志
pm2 logs myedgetunnel --err

# 手动启动查看详细错误
cd /opt/myedgetunnel
node server.js
```

### 无法访问

检查清单：
1. ✓ 应用是否运行: `pm2 status`
2. ✓ 端口是否监听: `netstat -tunlp | grep 8080`
3. ✓ 防火墙是否允许: `ufw status`
4. ✓ 查看日志: `pm2 logs myedgetunnel`

### 端口被占用

```bash
# 查看占用进程
netstat -tunlp | grep 8080

# 杀死进程
kill -9 <PID>

# 或修改端口
nano /opt/myedgetunnel/.env
```

## 性能优化建议

### 1. 使用集群模式

编辑 `ecosystem.config.cjs`：
```javascript
instances: 'max',  // 使用所有 CPU 核心
```

### 2. 配置 Nginx 反向代理

```bash
# 安装 Nginx
apt-get install nginx

# 复制配置
cp /opt/myedgetunnel/nginx/edgetunnel.conf /etc/nginx/sites-available/
ln -s /etc/nginx/sites-available/edgetunnel.conf /etc/nginx/sites-enabled/

# 重启 Nginx
systemctl restart nginx
```

配置后可通过 http://69.5.7.220 访问（无需端口号）。

### 3. 配置 HTTPS

```bash
# 安装 Certbot
apt-get install certbot python3-certbot-nginx

# 获取证书（需要域名）
certbot --nginx -d your-domain.com
```

### 4. 启用日志轮转

```bash
pm2 install pm2-logrotate
pm2 set pm2-logrotate:max_size 10M
pm2 set pm2-logrotate:retain 10
```

## 技术架构

```
用户请求
    ↓
Nginx (可选，80/443端口)
    ↓
Node.js Server (8080端口)
    ↓
Worker 适配层
    ↓
原始 Cloudflare Worker 代码
    ↓
KV 存储模拟 (内存)
```
